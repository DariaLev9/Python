# UI автотесты (PageObject + Allure)

Проект содержит UI автотесты, написанные с использованием следующих инструментов:

- Python
- Selenium WebDriver
- Pytest
- Page Object Pattern
- Allure Report

Тесты проверяют работу веб-приложений:

- калькулятор с задержкой выполнения операций
- интернет-магазин (добавление товаров и проверка итоговой стоимости заказа)


# Структура проекта

lesson_10

calculator_page.py — страница калькулятора  
login_page.py — страница авторизации  
inventory_page.py — страница товаров  
cart_page.py — страница корзины  
checkout_page.py — страница оформления заказа  

test_calculator.py — тест калькулятора  
test_shop.py — тест интернет-магазина  


# Page Object классы

Файлы страниц содержат методы для взаимодействия с элементами интерфейса.

- `calculator_page.py` — работа со страницей калькулятора  
- `login_page.py` — авторизация пользователя  
- `inventory_page.py` — страница товаров  
- `cart_page.py` — страница корзины  
- `checkout_page.py` — оформление заказа  


# Установка зависимостей

Перед запуском тестов необходимо установить зависимости:

pip install pytest
pip install selenium
pip install allure-pytest



# Установка Allure

Для генерации отчетов необходимо установить Allure

MacOS:
brew install allure



# Запуск тестов

Перейдите в папку проекта и выполните:

python3 -m pytest



# Запуск тестов с формированием Allure отчета

Чтобы сохранить результаты выполнения тестов для Allure, необходимо запустить pytest с параметром:

python3 -m pytest --alluredir=allure-results

После выполнения тестов в проекте появится папка:
allure-results

В этой папке сохраняются результаты выполнения тестов.


# Просмотр Allure отчета

Чтобы сформировать и открыть отчет Allure в браузере, выполните команду:

allure serve allure-results


Команда автоматически:

1. генерирует отчет
2. запускает локальный сервер
3. открывает отчет в браузере


# Папки Allure

Папки с результатами и отчетом не добавляются в репозиторий.

Их необходимо добавить в `.gitignore`:
allure-results/
allure-report/
